유지 보수

페이지 생성 시간을 최적화하기 위해 Piwigo는 캐시된 정보를 사용합니다. 예를 들어, 각 페이지에서 각 앨범에 포함된 사진 수를 계산하는 대신 이 정보는 데이터베이스에 저장됩니다. 이론적으로 이 정보는 항상 정확해야 하지만 때로는 오류가 발생하고 캐시된 정보가 최신 정보가 아닌 경우가 있습니다.

일부 정보는 시간이 지날수록 유용성이 떨어집니다. 데이터베이스에서 이 쓸모 없는 정보를 삭제하면 디스크 공간이 절약됩니다.

  • 이용하지 않는 태그 삭제
  • 앨범정보 업데이트: 각 앨범에 대해 다음 정보를 확인하고 필요한 경우 업데이트합니다: 상위 앨범 목록, 사진 수, 마지막 사진 날짜, 형제 앨범 중 위치, 모든 앨범 중 위치. 이 작업은 대표적인 사진 일관성도 확인합니다.
  • 사진정보 업데이트: 각 사진에 대해 업데이트된 정보는 전체 파일 경로, 평균 속도입니다. 경고, 혼동하지 마십시오, 메타 데이터 정보는 관리자 » 도구 » 동기화에서 동기화되어야 합니다, 또는 단일 사진의 수정 화면에 나타납니다. ( 사진 예시를 통하여).
  • 데이터베이스 복구와 최적화:데이터베이스의 각 테이블에서 재주문, 복구 및 최적화 작업을 수행합니다.
  • 무결성 검사 초기화
  • 히스토리 상세내역 삭제: 기록 테이블에서 모든 줄을 삭제합니다. 화면 관리자 »도구 » 히스토리은 더 이상 과거 내역에 대한 정보를 표시하지 않습니다. 경고: 데이터를 복구할 방법이 없이 모든 데이터가 손실됩니다.
  • 히스토리 요악 삭제: 방문 내역에 대한 모든 요약 정보를 삭제합니다. 이 요약은 히스토리 상세 내역에서 계산됩니다.
  • 세션 삭제: 만료된 사용자 세션을 삭제합니다.
  • 사용되지 않은 알림 피드 기록 세부 정보 삭제
  • 검색기록 삭제
  • 컴파일 템플릿 삭제
